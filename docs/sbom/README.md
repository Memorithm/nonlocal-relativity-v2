# SBOM — Software Bill of Materials

[`scirust.cdx.json`](scirust.cdx.json) is a **CycloneDX 1.5** SBOM for the
`scirust` facade crate and its full transitive dependency closure. CycloneDX
JSON is consumed directly by industrial supply-chain scanners (OWASP
Dependency-Track, Anchore Grype, etc.).

## Provenance

- **Generated by** `cargo cyclonedx` from the committed `Cargo.lock` — the
  single source of truth for resolved dependency versions.
- **Reproducible**: the BOM timestamp is pinned to the commit date via
  `SOURCE_DATE_EPOCH` and no random serial number is emitted, so the same
  source tree yields a byte-identical SBOM (consistent with the project's
  determinism guarantee).

## Regenerate

```sh
cargo install cargo-cyclonedx --locked   # once
./scripts/generate-sbom.sh
```

## Freshness

- CI (`.github/workflows/ci.yml`, job `sbom`) regenerates the SBOM on every
  build and uploads it as an artifact.
- The release workflow (`.github/workflows/release.yml`) regenerates it on
  each `v*` tag and attaches it to the GitHub Release.

This committed snapshot is a convenience for browsing; the authoritative SBOM
for a given version is the one attached to that version's release.
